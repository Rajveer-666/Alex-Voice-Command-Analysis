# 🎙️ Alexa Wake Word Detection — Voice Model Analysis

A machine learning pipeline for **Alexa keyword spotting** using audio features extracted from real voice recordings. Three classifiers are trained and evaluated on MFCC-based features to detect whether the wake word "Alexa" was spoken.

## 📌 Project Overview

This project addresses the binary classification problem of **wake word detection** — determining whether an audio clip contains the word "Alexa". It uses:

- **Real positive samples** from the `aanhari/alexa-dataset` (Kaggle)
- **Speech-based hard negatives** synthetically generated by corrupting real Alexa recordings (pitch shift, time stretch, reversal + noise, chunk shuffle)
- **MFCC feature extraction** via `librosa` for a compact 80-dimensional audio fingerprint
- **Three ML classifiers**: Logistic Regression, Random Forest, and SVM (RBF Kernel)

## 🗂️ Dataset

| Property         | Detail                                      |
|-----------------|---------------------------------------------|
| Source          | [`aanhari/alexa-dataset`](https://www.kaggle.com/datasets/aanhari/alexa-dataset) on Kaggle |
| Format          | `.wav` audio files @ 16kHz, mono            |
| Positive class  | 221 real "Alexa" wake word recordings       |
| Negative class  | 221 hard negatives (speech-corrupted audio) |
| Total samples   | 442                                         |
| Train / Test    | 353 / 89 (80/20 stratified split)           |

### Hard Negative Generation Strategy

Pure noise negatives are trivially separable from speech. Instead, **4 speech-based corruption techniques** are applied to real Alexa recordings to create realistic hard negatives:

1. **Extreme pitch shift** (±6 semitones) — distorts speech formants
2. **Time stretch** to a completely different rate
3. **Reverse + noise combo** — unrecognizable but speech-like MFCCs
4. **MFCC-domain chunk shuffle** — scrambles temporal order

---

## 🔧 Feature Engineering

Each audio clip is processed into an **80-dimensional MFCC feature vector**:

```
Audio (.wav) → Load @ 16kHz → Pad/Truncate to 1 second
    → 40 MFCCs → [mean across time (40) + std across time (40)]
    → Feature vector of shape (80,)
```

Features are then normalized using `StandardScaler` (mean ≈ 0, std ≈ 1) before training.

---

## 🤖 Models

### Model 1 — Logistic Regression
- Type: Linear, Parametric, Probabilistic
- Regularization: L2, C=0.01 (strong penalty to combat overfitting)
- Solver: `lbfgs`, max_iter=1000

### Model 2 — Random Forest Classifier
- Type: Ensemble, Non-parametric
- Handles non-linear feature interactions natively
- No scaling required (tree-based)

### Model 3 — Support Vector Machine (RBF Kernel)
- Type: Kernel-based, margin maximization
- Uses `CalibratedClassifierCV` for probability outputs
- Requires feature scaling (StandardScaler applied)

---

## 📊 Evaluation Metrics

Each model is evaluated on:
- **Accuracy** (test set + 5-fold cross-validation)
- **Precision**, **Recall**, **F1-Score**
- **AUC-ROC** curve
- **Confusion Matrix**

## 🚀 Getting Started

### Requirements

```bash
pip install librosa scikit-learn numpy pandas matplotlib seaborn soundfile
```

### Run on Kaggle

1. Open the notebook: [Alexa Voice Model Analysis](https://www.kaggle.com/code/rajveer1035/alexa-voice-model-analysis)
2. Add the dataset: `aanhari/alexa-dataset`
3. Click **Run All**

### Run Locally

```bash
git clone https://github.com/rajveer1035/alexa-voice-model-analysis.git
cd alexa-voice-model-analysis

# Download dataset from Kaggle
kaggle datasets download -d aanhari/alexa-dataset
unzip alexa-dataset.zip -d data/

# Run the notebook
jupyter notebook Alexa_Voice_Model_Analysis.ipynb
```
